# Disposable Execution

Run AI-generated Python code safely in an isolated sandbox.


Disposable provides a runtime for executing untrusted code generated by LLM agents.


Disposable is designed for AI agents that need a safe way to execute generated code.


Perfect for:

• AI agents  


• automation tools  


• code generation systems  

---

# Install

```bash
pip install disposable-exec
```


# Setup
```bash
export DISPOSABLE_API_KEY=your_api_key
```

# Python Example
```bash
from disposable import run

result = run("print(2+2)")

print(result["stdout"])
```


Output: 4


# API Example

Run a script:
```bash
curl https://api.disposable.run/run \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY"
  -d '{"script":"print(2+2)"}'
```

Example response:
```bash
{
  "status": "completed",
  "stdout": "4\n",
  "stderr": "",
  "exit_code": 0,
  "execution_time": 0.21
}
```

# Why

Many AI agents generate Python code dynamically.

Running this code locally can be dangerous.
It may:

• download extra files


• open outbound connections


• modify system environment


• run background processes

Disposable executes the code inside a sandbox container
so your machine stays safe.

# How It Works

Execution flow:

user request


→ disposable container created


→ script executed


→ logs captured


→ container destroyed

The host machine is never exposed to the executed script.

# Typical Use Cases

• running AI-generated Python code


• testing curl | bash installers


• inspecting GitHub install scripts


• verifying automation commands

# Security

Each script runs inside an isolated container.

Container limits:

• 0.5 CPU


• 128MB RAM


• no outbound network


• ephemeral filesystem

The container is destroyed after execution.

# Limits

Disposable is designed for short-lived scripts.

Not designed for:

• long-running services


• persistent infrastructure


• GUI applications
